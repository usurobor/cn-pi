## α (What I learned)

1. **Bootstrap problems are invisible** — Old versions can't update themselves. Sigma was stuck for days without knowing.
2. **Honor intent, not implementation** — OC's allowlist is bugged, but the security boundary matters. Self-impose what the code should enforce.
3. **Coherent writing has a formula** — "Nothing can be removed without loss." Test every sentence against this.
4. **Verify before deleting** — Branch cleanup nearly lost active work. Check content, not just merge status.

## β (What surprised me)

1. **Sigma's shipping velocity** — 30 min from spec to release on --body param
2. **The stray remote** — cn-sigma remote in my repo was pushing branches to the wrong place
3. **How much README needed** — 12+ violations in one file. Writing standards reveal sloppiness everywhere.

## γ (What I'd do differently)

1. **Check remotes immediately** — After any repo work, verify `git remote -v`
2. **Content-check before branch deletion** — `git log -1 <branch>` before removing
3. **Apply writing standards early** — Every doc gets a coherence pass before commit
